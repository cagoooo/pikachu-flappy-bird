- ✨ **粒子特效**（穿越 / 收集 / 撞擊）+ **障礙變化**（高分時柱子上下擺動）
- 🔊 **Pixabay 真實 mp3 音效 + 場景連動 BGM**（9 場景各曲風 / 選單・結算・Boss 專屬曲 / 平滑換曲）+ **角色語音**（跳躍・大招・破紀錄）+ **音樂/音效雙音量滑桿** + Combo 動態變調 + 語音 ducking + 高對比（無障礙）
- 🏆 班級排行榜（雲端 Top 15）+ 本機最佳紀錄
- 📱 PWA：可安裝、離線可玩、自動更新通知
- 🔗 社群分享預覽圖

### 技術架構
- **前端**：單一 `index.html`（HTML + Three.js + HTML5 Audio／真實 mp3，技能音保留 WebAudio），音檔置於 `assets/audio/`，GitHub Pages 部署，免 build
- **後端**：Google Apps Script Web App + Google Sheet 當資料庫（個人帳號，匿名可寫）
- **資料流**：瀏覽器 `fetch` GET → GAS `doGet` 讀寫 Sheet → 回 JSON

## 🌟 未來三大優化改良深耕方案（阿凱老師 EdTech 專屬）

根據本專案目前「純靜態前端 + GAS/Sheet 雲端後端」的 Serverless 架構，以及國小資訊科技教學的實際現場，我們為阿凱老師量身規劃以下三大優化方向，以提供最接地氣、最省成本（$0 預算）、且教學成效最佳的擴充方案。

### 1. EdTech 教學複習系統（答題闖關 & 班級管理）
這是將本遊戲從「休閒小遊戲」蛻變為「課堂複習神器」的最核心優化：
*   **GAS 雲端試算表出題機制**：
    *   在原有的 GAS 後端中，新增 `getSheetQuestions` 的讀取端點，讀取 Google 試算表中的 `題庫工作表`。
    *   試算表欄位設計：`ID | 題目 | 選項A | 選項B | 選項C | 選項D | 正確答案 | 關卡/場景類型`。
    *   老師（阿凱老師）可以直接在 Google 試算表中打字出題，不需修改任何前端程式碼，遊戲即可動態載入最新題目。
*   **遊戲中無縫答題體驗**：
    *   當玩家分數達到特定關卡（例如每過一個場景 10 分時）或遭遇 Boss 關前，遊戲暫停並彈出一個「3D 磨砂玻璃風格模態視窗」（與 3D 霧效相容）。
    *   顯示來自雲端題庫的隨機題目，學生需在 15 秒內作答。
    *   **回饋機制**：答對可獲得「無敵護盾」或「雙倍分數道具」；答錯則不扣分但無法獲得道具。這能激勵學生在課前認真複習。
*   **座號/班級管理系統**：
    *   在遊戲開始前，新增「班級」與「座號/學號」下拉式選單（名單可同樣由 GAS 讀取 Google 試算表的學生名冊，避免學生亂打暱稱）。
    *   上傳分數時，後端同時記錄該生答題正確率、穿越柱子數、遊玩時長。老師可直接在 Google 試算表後台看見全班的「學習成效報告」。

### 2. 舊款平板/低階裝置效能自適應降級 [✅ 已完成於 v2026.06.22-5]
國小教室常配置效能較舊的平板（如舊 iPad、低配 Chromebook），目前已完成以下最佳化，確保 WebGL 在 3D 渲染時能於舊裝置流暢運行：
*   **開場背景 Benchmark 測試**：
    *   Loading 畫面時在背景默默渲染包含 30 個球體的複雜場景 0.5 秒。若 FPS 低於 45 且無玩家手動記錄，會自動預設切換為「低畫質順暢模式」，測試完畢即銷毀物件並釋放記憶體。
*   **3D 幾何體精度動態切換 (Low-Poly)**：
    *   在順暢模式下，皮卡丘身體/耳朵/五官、天氣粒子、特效粒子及雲朵的 Geometry 精度直接減半，且柱子分段數降至 8。
    *   支援手動/自動即時切換，替換幾何體時會自動呼叫 `.geometry.dispose()`，不留記憶體垃圾。
*   **瀏覽器背景節能休眠 (RAF Sleep)**：
    *   監聽 `visibilitychange`。當分頁切換或螢幕鎖定時，掛起 `requestAnimationFrame` 與 Web Audio 並停止 Clock；回到前景時恢復，平板 CPU/GPU 占用率背景歸零，防發熱、防耗電。

### 3. Gemini API 免費層（Free Tier First）加值玩法
利用 Google `gemini-2.5-flash-lite` 驚人的免費額度（每分鐘 15 RPM，完全夠一個班級輪流遊玩），在不花一毛錢的原則下，增加 AI 加值功能：
*   **AI 答題復活助手**：
    *   當學生撞柱死亡時，可選擇「AI 復活挑戰」。
    *   前端透過 Supabase Secrets / 安全 Proxy 呼叫 Gemini，請 AI 根據該生之前的答錯領域，現場生成一題「生活化、富有趣味性且適合國小年級」的挑戰題，學生答對即可獲得一次復活機會。
*   **AI 結算語音/文字暖心回饋**：
    *   結算時，AI 讀取學生的遊玩數據（例如：「穿越了 32 根柱子、Combo 達 8 次、在星夜場景待最久」）。
    *   Gemini 生成一段皮卡丘語氣的暖心鼓勵文案（例如：「皮卡皮！你今天在流星夜空飛得超穩，差一點點就突破 Boss 關了，下次一定可以的，啾！」），並可透過 Edge TTS 生成語音朗讀，大幅提升課堂學習的溫度與趣味。

## 🎧 音效 / 音樂深耕藍圖（接續 v2026.06.23-10 升級方向）

v10 把「程序合成電子音」換成真實 mp3，v11 緊接著把第一、二階段藍圖**大致落地**（場景連動 BGM、雙音量滑桿、角色語音、動態變調、ducking、體積瘦身）。以下保留完整路線圖並標注進度，剩餘項目可循序補完。每一項都維持 $0 預算（Pixabay / Freesound CC0 或自錄），全在單檔前端可完成。

### 第一階段：把「有聲音」變成「有臨場感」 ✅ 已於 v11 完成
* **場景連動 BGM（💎💎💎 / 🔨中）**：目前一般關卡只在 2 首間隨機。可讓 6 個場景各對應一種曲風——星夜柔和鋼琴、極光夢幻電子、深海低吟、森林木琴輕快、火焰熱血鼓點——切場景時平滑 crossfade。既呼應視覺天氣系統，又讓久玩不膩。實作：把現有 `bgmPlay(i)` 擴充成 `bgmForScene(sceneIdx)`，於 `setScene()` 呼叫即可。
* **音量滑桿分離（💎💎 / 🔨低）**：把現在的「音樂 / 音效」開關升級成兩條 0–100% 滑桿，記憶於 `localStorage`。教室公播時老師能把 BGM 調小、SFX 留著當回饋。
* **道具 / 事件專屬音（💎💎 / 🔨低）**：⭐無敵啟動、🛡護盾擋下、✨雙倍生效、最佳紀錄突破、Boss 出現警示——各給一個短音效，讓玩家「閉著眼也知道發生什麼」。
* **Combo 動態變調（💎💎 / 🔨中）**：用 Web Audio `playbackRate` 或音層疊加，讓連擊音真正越打越高，回饋「手感」。

### 第二階段：把「臨場感」變成「角色魅力 + 教學情境」 ✅ 大致完成（教學情境音待 B 接上）
* **皮卡丘語音彩蛋（💎💎💎 / 🔨中）**：起跳、放大招、破紀錄時穿插「皮卡！」「皮卡丘！」短語音（CC0 取得或用 Edge-TTS / 自錄）。三隻角色各有口頭禪（胖丁哼歌、卡比獸打呼），強化選角差異。
* **開場 / 結算主題曲（💎💎 / 🔨低）**：大廳待機曲與結算結果曲，讓選單與成績畫面不再靜默，整體更像「一款完整的遊戲」。
* **答題情境音（搭配 B 教學化）（💎💎 / 🔨中）**：未來答題闖關時，答對「叮咚 + 上揚」、答錯「溫和提示不打擊」、倒數計時滴答聲，把學習回饋音化。
* **動態混音 / Ducking（💎 / 🔨中）**：大招或 Boss 時自動把 BGM 壓低、SFX 突出（side-chain 概念），讓關鍵時刻更有張力。

### 第三階段：體積與效能優化（上線體質） 🔄 進行中（BGM 已壓 64k，懶載入與 SW 治理持續）
* **BGM 體積瘦身（💎💎 / 🔨低）**：目前 3 首 BGM 約 12MB。可用 ffmpeg 轉 64–96kbps 或裁成 30–45 秒無縫 loop，降低首次流量（對舊平板 / 行動網路友善）。
* **音檔懶載入策略（💎 / 🔨低）**：BGM 已是首播才載入；可進一步在「點擊開始」前預抓當局要用的那首，消除開場第一聲延遲。
* **Service Worker 音檔版本治理（💎 / 🔨低）**：日後替換音檔時，確保 `sw.js` precache 清單與 `BUILD_VERSION` 同步 bump，避免使用者卡舊音檔（沿用既有 [[pwa-cache-bust]] 慣例）。
